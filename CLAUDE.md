## Project Overview

A go-zero learning project — order API service that queries orders from MySQL and calls an upstream user gRPC service to enrich responses.

## Architecture

```
Request → Handler → Logic → OrdersModel (MySQL + Redis cache)
                         → UserRpc (gRPC client → ../user-rpc)
```

- **Module name**: `order` (see go.mod)
- **Framework**: go-zero v1.10.2 REST API
- **Database**: MySQL `db1.orders` table, accessed via sqlx model with Redis LRU cache
- **Upstream dependency**: `user-rpc` (local module at `../user-rpc`, replace directive in go.mod)

## Code generation commands

```bash
# Regenerate API code from .api definition (types, routes, handlers)
goctl api go -api order.api -dir .

# Regenerate DB model from live MySQL table
goctl model mysql datasource -url="root:root123456@tcp(127.0.0.1:3306)/db1" -table="orders" -dir="./model" -cache=true
```

## Build and run

```bash
# Build
go build -o order.exe .

# Run (listens on 0.0.0.0:8888)
./order.exe -f etc/order-api.yaml
```

Before starting the order service, ensure:
- MySQL is running on `127.0.0.1:3306` with database `db1`
- Redis is running on `127.0.0.1:6379`
- `user-rpc` service is running on `127.0.0.1:8080`

## API endpoint

```
GET /order/search?orderNo=xxx
```

Response includes order info (from `db1.orders`) and the user's `userName` fetched via `user-rpc.GetProfile(userId)`.

## .api tag conventions

| HTTP method | Use tag | Example |
|---|---|---|
| GET | `form` only, never mix with `json` | `form:"orderNo"` |
| POST JSON body | `json` | `json:"name"` |
| Response fields | `json` | `json:"orderInfo"` |

Mixing `json` and `form` on a GET request field causes `httpx.Parse` to fail with "field is not set".

## Model layer patterns

- `genesmodel_gen.go`: auto-generated by goctl. Contains `FindOne` (by PK), `FindOneBy*` (by unique keys), `Insert`, `Update`, `Delete`
- `ordersmodel.go`: embed `ordersModel` interface, add custom methods here (e.g., queries by non-unique indexes, joint indexes, paginated queries)
- Non-unique index queries should use `QueryRowsNoCacheCtx` — go-zero's cache is designed for PK/unique-key lookups
- Re-run `goctl model mysql datasource` when table schema changes; it overwrites `_gen.go` but preserves `ordersmodel.go`
